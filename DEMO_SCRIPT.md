# BenchPilot 105-second demo script

## 0:00–0:10 — The problem

**On screen:** BenchPilot landing view. Hold on the tagline and **Load zinc-air demo** button.

**Voiceover:** “Physical experiments rarely arrive as clean data. They arrive as photos, rough notes, and readings that seem to contradict each other. BenchPilot turns that mess into reproducible evidence—and a better next experiment.”

## 0:10–0:23 — Capture

**Action:** Select **Load zinc-air demo**. Show the apparatus photo/preview, zinc-air configuration, and raw note.

**Voiceover:** “This is a real zinc-air prototype: zinc anode, carbon and manganese-dioxide air cathode, acrylic binder, about 30% KOH, and no conventional separator. One run collapsed under a fan load. An earlier run recovered over time.”

## 0:23–0:37 — Structure

**Action:** Move to **Structure**. Scroll just enough to reveal provenance labels and measurements.

**Voiceover:** “GPT-5.6 turns the notes and image into a validated record. Notice the boundaries: user-reported facts, image observations, calculated results, unknowns, and safety considerations never blur together. Invalid model output cannot reach this interface.”

## 0:37–0:53 — Challenge

**Action:** Select **Challenge**. Focus on the competing explanation cards and one falsifier.

**Voiceover:** “Instead of agreeing with my first theory, Challenge My Interpretation proposes a small set of competing mechanisms. Each shows evidence for, evidence against, confidence, what remains unknown, and one observation that would falsify it.”

## 0:53–1:10 — Hypothesis Matrix wow moment

**Action:** Bring the Hypothesis Matrix fully into view. Add or activate the prepared discriminating measurement. Pause on the changed cell and evidence-shift explanation.

**Voiceover:** “Here is the key idea. Rows are hypotheses; columns are observations and planned measurements. A cell supports, contradicts, or does not distinguish. Add a new measurement, and the matrix updates—not with a new conclusion, but with a precise explanation of which hypothesis gained or lost support.”

## 1:10–1:25 — Test

**Action:** Select **Test** and highlight the first-ranked experiment, controls, expected outcomes, and safety note.

**Voiceover:** “BenchPilot ranks the next tests by information gained per unit effort. The top plan changes one variable, holds the others constant, specifies exact readings and stop conditions, and predicts a different result under each explanation.”

## 1:25–1:40 — Compare

**Action:** Select **Compare**. Trace the two lines on the voltage chart, then show configuration differences.

**Voiceover:** “Compare uses only validated measurements. The timeline and chart make the collapse and slow recovery unmistakable, while configuration differences and evidence shifts stay alongside the data.”

## 1:40–1:45 — Close

**Action:** Return focus to the workflow summary or report action; end on the BenchPilot mark.

**Voiceover:** “BenchPilot does not claim to prove the answer. It helps every experiment produce clearer evidence for what to test next.”

## Recording note

Use the seeded no-key path for the primary take. Record one optional live-analysis insert separately so a slow or unavailable API cannot interrupt the 105-second story.
